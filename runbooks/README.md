# runbooks/

Runbooks são **procedimentos operacionais**: passo a passo replicável para configurar, investigar ou executar tarefas comuns em SOC/Blue Team.

## O que entra aqui
- “Como habilitar e coletar logs do Windows”
- “Como instalar/configurar Sysmon”
- “Como filtrar eventos por Event ID e exportar”
- “Como analisar um alerta simples”
- “Como preparar o AD/Lab para gerar eventos”

## Padrão de runbook
Cada runbook deve ter:
1) Objetivo (1–2 linhas)
2) Pré-requisitos
3) Passo a passo numerado
4) Evidências (prints / comandos)
5) Resultado esperado
6) Troubleshooting rápido (erros comuns)

## Exemplo
- `ad-ds.md` → Active Directory Domain Services
